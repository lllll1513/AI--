## 顶级工作流一句话

AI漫剧/短剧不是“一个提示词生成一条片”，而是一个可控生产系统：

剧本结构化 -> 角色资产固定 -> 场景资产固定 -> 分镜脚本 -> 静帧/故事板 -> 逐镜头视频 -> 质检重生成 -> 配音音效 -> 自动剪辑 -> 精修交付。

## 核心原则

1. 资产先行，不要裸跑视频。
   角色、对手、场景、道具、武器、特效必须先成为可复用资产。视频模型只负责让资产动起来。

2. 一镜一任务，不要让模型一次拍完整片。
   每个镜头 4-8 秒最稳。复杂战斗优先拆成 5 秒片段。

3. Prompt 负责动作和镜头，不负责重复描述长相。
   角色长相靠参考图和素材 ID 锁定，文字只写当前镜头发生什么。

4. 名称必须严格绑定。
   角色名、场景名、武器名在剧本、分镜、素材库、提示词中必须完全一致，禁止同义词、简称、代称乱飞。

5. 故事板先行。
   先控制构图、景别、人物站位、镜头运动，再进入视频生成。只控构图可以明显降低场景漂移和画质掉线。

6. 质检必须前置。
   静态、跑偏、角色不像、动作弱、画面崩、参考图没用上，都不能进入最终合成。

7. 自动剪辑不是简单拼接。
   合格镜头才进入粗剪；爆点、铺垫、情绪、动作节奏要被标注，后续才能自动决定硬切、叠化、慢镜、音效点。

## 标准生产阶段

### 1. 项目设定

必须确定：
- 片名
- 横屏/竖屏
- 目标时长
- 类型：玄幻、古风、都市、情感、科幻、悬疑、战斗等
- 视觉风格
- 受众和平台
- 是否需要连续剧集

输出：
- 项目总控文档
- 世界观
- 风格圣经
- 角色清单
- 场景清单

### 2. 剧本

剧本不要只写剧情梗概，要输出可执行结构：
- 场次
- 镜头意图
- 角色动作
- 台词
- 情绪曲线
- 爆点
- 转场
- 声音提示

短剧结构建议：
- 0-5 秒：钩子
- 5-20 秒：冲突建立
- 20-45 秒：升级
- 45-70 秒：爆点
- 70-90 秒：悬念或收束

### 3. 角色资产

每个重要角色至少需要：
- 全身设定图
- 三视图或四视图
- 面部特写
- 表情参考
- 武器/道具
- 常用服装
- 禁忌描述
- 视频生成时的短标签

角色资产规则：
- 主角和反派必须优先高质量生成。
- 杂兵可以批量化，但也要有统一阵营风格。
- 同一角色在项目内只能有一个标准名称。
- 所有镜头引用角色时都用标准名称。

### 4. 场景资产

场景不只是背景图，而是镜头空间。

每个场景建议包含：
- 远景空间图
- 中景战斗区/行动区
- 近景细节
- 光线和天气
- 损毁状态
- 特效状态

战斗项目要额外准备：
- 战场远景
- 军阵/杂兵层
- 地面破坏
- 烟尘
- 火光
- 武器碰撞火星
- 冲击波

### 5. 分镜

分镜必须包含：
- 镜头编号
- 时长
- 景别
- 构图
- 角色站位
- 动作起点
- 动作终点
- 镜头运动
- 台词
- 音效
- 使用素材

战斗分镜建议拆法：
- 远景建立战场
- 中景交锋
- 近景武器碰撞
- 特写眼神/怒吼
- 慢动作爆点
- 反应镜头
- 结果镜头

### 6. 静帧/故事板

两种路线：

首尾帧模式：
- 最适合明确动作起点和终点。
- 适合挥砍、转身、冲刺、爆炸、倒地。
- 画面更可控，但准备成本高。

参考图模式：
- 适合角色一致性和风格一致性。
- Seedance 2.0 可传多张参考图。
- 需要 Prompt 明确“使用哪张参考图作为哪类信息”。

故事板关键：
- 先让画面像，再让画面动。
- 只要静帧人物不像，视频阶段很难救回来。

### 7. 视频生成

Seedance 2.0 方向要点：
- 支持文生视频、图生视频、首尾帧、多模态参考、视频参考、音频参考、生成有声视频、返回尾帧、视频延长等能力。
- 多参考图一般适合角色和风格锁定。
- 首尾帧适合严格控制动作变化。
- 时长建议从 5 秒起步，复杂镜头不要贪长。
- Prompt 不要过长，否则信息分散。

视频 Prompt 推荐结构：

```text
Duration: 5s. 16:9 cinematic action shot.

吕布从破碎战场中央猛然踏前，方天画戟由右下向左上横扫，三名强敌同时后撤格挡，兵刃碰撞爆出赤红火星。镜头低机位跟随戟刃运动，尘土和披风被冲击波掀起。

Dialogue/lip sync: 吕布低吼：“挡我者，死。”
```

重点：
- 写动作路径。
- 写镜头运动。
- 写物理反馈。
- 写情绪和节奏。
- 台词单独标注。

### 8. 质检

每个片段必须检查：
- 是否明显运动，不是静态图漂移。
- 主角是否像参考图。
- 对手是否存在且站位正确。
- 武器是否正确。
- 镜头是否符合分镜。
- 是否有肢体崩坏。
- 是否有画面糊、脸崩、穿模。
- 是否有台词口型或声音问题。

质检结果：
- 通过：进入粗剪。
- 小问题：可进入备选。
- 大问题：重生成。

### 9. 配音与声音

声音层级：
- 人声台词
- 旁白
- BGM
- 环境声
- 打击音
- 转场音
- 低频冲击

AI短剧最容易廉价的地方不是画面，而是声音。声音要作为独立资产管理。

### 10. 自动剪辑

自动剪辑分三层：

基础拼接：
- 按镜头顺序合成。
- 加字幕、片头、片尾、BGM。

选择性粗剪：
- 只合成通过质检的镜头。
- 自动跳过失败片段。
- 导出重生成清单。

审美剪辑：
- 根据节奏标签决定硬切、叠化、慢动作、音效点。
- 根据情绪曲线安排音乐起伏。
- 根据爆点镜头做二次强调。

## 我们工作台的目标

最终不是做一个“提示词仓库”，而是做一个“AI漫剧制片系统”：

- 项目有记忆。
- 角色有资产。
- 分镜有状态。
- 视频有质检。
- 剪辑有规则。
- 失败有原因。
- 每个项目结束后反哺模板库。

## 黑神话吕布类项目的最佳实践

这类项目难点：
- 角色气场要强。
- 对手要配得上。
- 战场要大。
- 动作要有力量感。
- 不能像普通队长或营长。

推荐结构：
- 第一镜：战场远景，魔神压场。
- 第二镜：三名强敌登场，各有气势。
- 第三镜：杂兵冲阵，被一击震退。
- 第四镜：一挑三正式开打。
- 第五镜：方天画戟破防。
- 第六镜：近景怒吼或台词。
- 第七镜：爆点斩击。
- 第八镜：战场沉寂，留下悬念。

素材优先级：
1. 吕布魔神形态
2. 三名强敌
3. 方天画戟
4. 战场远景
5. 杂兵军阵
6. 火星/冲击波/尘土/披风风压
7. 战鼓/BGM/低频音效

## 后续知识升级方向

- 跟踪 Seedance、Kling、Veo、Wan 的最新能力差异。
- 收集优秀成片拆解：镜头长度、构图、动作密度、台词密度。
- 建立失败案例库：静态、跑偏、崩脸、武器错、动作弱。
- 建立提示词模板库：打斗、情绪、转场、追逐、奇观、对话。
- 建立项目复盘机制：每个项目结束后更新模板。
