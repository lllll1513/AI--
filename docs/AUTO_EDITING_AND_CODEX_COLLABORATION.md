# 自动剪辑与 Codex 协作方案

## 现在能自动剪辑到什么程度

当前地基已经具备“基础自动剪辑”能力：

- 按镜头顺序拼接所有已生成视频片段。
- 自动生成片头卡和片尾卡。
- 自动生成 SRT 字幕，并可烧录到视频里。
- 支持简单转场：硬切、叠化、淡入淡出、擦除、滑动、圆形展开。
- 支持叠加 BGM，并自动按最终视频长度截断。
- 每个镜头可以单独预览，最后再合成完整视频。

这已经可以满足“每个镜头生成 5 秒，后期自动拼成一条片子”的基础流程。

## 现在还不算真正智能剪辑的地方

它还不会主动判断：

- 哪个生成片段更好，哪个应该废弃。
- 某个镜头是否静态、动作弱、角色跑偏。
- 哪些镜头应该缩短、延长、调换节奏。
- 哪里应该压音乐点、加冲击音、做慢动作或硬切。
- 哪个片段适合进最终版，哪个只适合当备选素材。

所以它现在更像“自动拼接器”，还不是“电影剪辑师”。

## 我们要补的自动剪辑中枢

第一层：素材可用性检查
- 检查每个镜头是否有视频。
- 检查视频文件是否存在、能打开、时长是否合理。
- 检查缺失素材：首帧、尾帧、参考图、视频、字幕、BGM。

第二层：镜头质量检查
- 角色是否像参考素材。
- 动作是否明显，不接受纯静态图漂移。
- 镜头是否符合分镜描述。
- 画面是否有严重崩坏、肢体错误、脸部错误。
- 参考图是否真的被使用。

第三层：剪辑节奏决策
- 自动识别开场、铺垫、爆点、收束。
- 战斗镜头优先硬切和短转场。
- 情绪镜头允许慢节奏和叠化。
- 高燃镜头自动标记为重点片段。
- 低质量片段进入“待重生成”，不进入最终合成。

第四层：声音与字幕
- 台词按镜头时长自动分配。
- BGM 自动压低音量，不盖过台词。
- 打斗镜头标记需要音效：兵器、冲击、风声、爆裂、低频。
- 后续增加 TTS、旁白、角色配音、音效素材库。

第五层：最终出片
- 生成粗剪版。
- 生成精剪建议清单。
- 生成可下载素材包：视频片段、字幕、音频、封面、工程说明。
- 后续可导出剪映/PR 可用的素材结构。

## Codex 和工作台怎么配合

工作台负责执行：
- 保存项目、角色、镜头、素材和任务。
- 调用 Seedance、Kling、Veo、图片模型、文本模型。
- 批量生成图片、视频、字幕和合成片。
- 保存每次生成结果和历史版本。

Codex 负责把控：
- 根据你的参考视频、素材和审美目标改剧本、分镜和提示词。
- 判断哪些生成结果跑偏，应该怎么重生成。
- 调整工作台代码和流程，让它越来越贴近你的生产方式。
- 帮你把黑神话吕布这种项目拆成角色包、场景包、动作包、镜头包。
- 后续可以持续帮你维护这个工作台，而不是每次从零开始。

你的角色：
- 给目标：想做什么片子、参考什么效果、横屏还是竖屏。
- 给素材：人物图、三视图、场景、对手、风格参考、视频参考。
- 做审美确认：哪些效果你喜欢，哪些你觉得不够。

## 黑神话吕布项目的推荐自动剪辑流程

1. 建项目：横屏电影短剧，16:9。
2. 导入素材包：吕布、三名强敌、杂兵、战场、武器、特效。
3. 生成分镜：每个镜头 5 秒左右。
4. 生成首尾帧或参考图。
5. 逐镜头生成视频片段。
6. 自动质检：静态、跑偏、崩坏、动作弱的片段直接标红。
7. 只把合格片段进入粗剪。
8. 自动合成粗剪版。
9. Codex 根据粗剪结果继续改分镜、提示词和重生成策略。

## 下一步开发顺序

优先级最高的是“自动剪辑前置质检”，因为它能直接解决你之前最在意的问题：图不动、乱生成、没有按素材走。

建议下一步做：

1. 在预览页增加“自动剪辑检查”面板。
2. 每个镜头显示：视频状态、素材完整度、质检结果、是否进入最终合成。
3. 增加“只合成合格镜头”的开关。
4. 增加“标记重生成原因”：静态、跑偏、动作弱、角色不像、画面崩。
5. 后续再接 AI 视觉模型做自动判断。
