# AI短剧工作台全自动化蓝图

目标：让 Codex 搭配 AI短剧工作台，尽量自动跑通从素材到成片的完整流程。

## 自动化总流程

```text
输入素材/参考视频/想法
  -> 项目初始化
  -> 剧本生成或导入
  -> 角色/场景/道具抽取
  -> 素材资产库创建
  -> 分镜拆分
  -> 故事板/首尾帧/参考图生成
  -> 视频提示词生成
  -> 逐镜头视频生成
  -> 自动质检
  -> 失败镜头重生成
  -> 配音/音效/BGM
  -> 自动粗剪
  -> 人审精修
  -> 最终导出
```

## 工作台要新增的核心模块

### 1. 项目模板

模板类型：
- 横屏电影短剧
- 竖屏短剧
- 漫剧
- 角色设定集
- 战斗短片
- 产品/口播短片

每个模板内置：
- 默认比例
- 镜头时长策略
- 分镜结构
- 提示词风格
- 字幕样式
- 质检规则

### 2. 素材资产库

资产类型：
- 角色
- 对手
- 杂兵
- 场景
- 武器
- 道具
- 特效
- 音效
- BGM
- 参考视频

每个资产需要：
- 标准名称
- 类型
- 描述
- 文件
- 使用限制
- 是否核心资产
- 关联镜头

### 3. 分镜工作流画布

不依赖 LibTV，项目内自建。

节点类型：
- 剧本节点
- 角色节点
- 场景节点
- 分镜节点
- 首帧节点
- 尾帧节点
- 参考图节点
- 视频节点
- 质检节点
- 配音节点
- 剪辑节点

一条镜头链路：

```text
分镜 -> 素材绑定 -> 静帧 -> 视频 -> 质检 -> 合成
```

### 4. 逐镜头 5 秒生成模式

规则：
- 默认每镜 5 秒。
- 情绪镜头可 4-6 秒。
- 战斗镜头优先 4-5 秒。
- 建立镜头可 6-8 秒。
- 复杂动作拆成多个镜头。

界面需要：
- 每镜生成按钮
- 批量生成按钮
- 失败原因
- 重生成原因
- 是否进入最终合成

### 5. 自动质检

第一阶段：规则质检
- 文件存在
- 时长合理
- 是否有视频
- 是否有首尾帧或参考图
- 是否有绑定角色
- 是否有视频提示词

第二阶段：AI质检
- 画面是否静态
- 角色是否像
- 武器是否正确
- 动作是否明显
- 镜头是否符合描述
- 是否崩脸/崩手/穿模

第三阶段：审美质检
- 是否有电影感
- 是否有动作力量
- 是否有节奏
- 是否值得进入最终片

### 6. 自动剪辑

粗剪规则：
- 只合成通过质检的镜头。
- 按镜头顺序排列。
- 片头片尾可配置。
- 字幕自动烧录。
- BGM 自动混音。
- 转场根据镜头标签决定。

剪辑标签：
- 开场
- 铺垫
- 情绪
- 动作
- 爆点
- 反应
- 收束
- 悬念

战斗剪辑规则：
- 动作镜头优先硬切。
- 爆点前可短暂停顿。
- 武器碰撞点配打击音。
- 慢动作只给最强一两个镜头。

### 7. 配音音效

需要补：
- 台词 TTS
- 旁白 TTS
- 角色音色管理
- BGM 上传
- 音效素材库
- 自动音量平衡

短期策略：
- 先支持上传 BGM 和音效。
- 台词先用字幕管理。
- 后续接 TTS 供应商。

### 8. 失败复盘

每次失败记录：
- 镜头编号
- 使用模型
- 使用素材
- 提示词
- 失败类型
- 重生成建议
- 是否已解决

失败类型：
- 静态
- 跑偏
- 角色不像
- 动作弱
- 画面崩
- 武器错误
- 场景错误
- 台词错误
- 生成失败

## Codex 的工作方式

Codex 不只是写代码，而是担任：
- 导演
- 编剧
- 美术指导
- 分镜师
- 剪辑策略师
- 工作流工程师
- 质检员

工作台负责执行；Codex 负责判断和改进。

## 最小可行版本

第一版先做：

1. 预览页增加“自动剪辑检查面板”。
2. 每个镜头显示素材完整度和视频状态。
3. 支持“进入最终合成 / 排除”。
4. 支持标记失败原因。
5. 合成时只使用进入最终合成的镜头。

这一步完成后，工作台就从“能拼视频”升级成“能筛选后再剪”。

并行准备：

- 建立参考视频逐帧审阅流程。
- 对教程视频、优秀成片、失败案例抽帧和记录。
- 把审阅结果转成项目模板、质检规则和剪辑策略。

## 第二版

1. 增加资产库页面。
2. 增加镜头绑定素材。
3. 增加项目模板。
4. 增加 AI 质检接口。
5. 增加重生成建议。

## 第三版

1. 增加可视化工作流画布。
2. 增加 TTS/音效/BGM 管线。
3. 增加自动粗剪策略。
4. 增加导出剪映/PR 素材包。

## 费用策略

默认优先本地和已有供应商。

LibTV、Flova、第三方平台只作为可选外部执行器，不默认触发付费任务。

任何可能花钱的动作必须：
- 先展示计划。
- 用户确认。
- 再执行。
- 执行后记录成本和结果。
