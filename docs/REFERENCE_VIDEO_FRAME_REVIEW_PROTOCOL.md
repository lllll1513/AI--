# 参考视频逐帧审阅流程

目标：把教程视频、优秀成片、失败案例拆成可复用知识，而不是只看标题和口播。

## 当前访问限制

用户提供的抖音短链在命令行环境中需要新鲜浏览器状态，无法直接抓取视频文件。B站短链也出现访问限制。

为了不假装看完，后续采用两种方式：

1. 用户把教程视频下载到本地文件夹。
2. Codex 使用本项目脚本抽帧、场景切分、运动检测、联系表审阅。

## 推荐放置目录

```text
D:\Codex\工作项目\AI短剧+ 漫剧\reference-videos\inbox
```

支持格式：
- `.mp4`
- `.mov`
- `.mkv`

## 审阅脚本

```powershell
.\scripts\review-video.ps1 -VideoPath "D:\path\to\video.mp4"
```

输出目录：

```text
video-review\<视频名>\
  sampled-frames\      # 按固定频率抽帧
  scene-frames\        # 按画面变化抽关键场景帧
  reports\
    metadata.json
    motion-report.json
    contact-sheet.jpg
```

## 逐帧审阅看什么

教程类视频：
- 工具链是什么。
- 每一步输入是什么。
- 每一步输出是什么。
- 哪些步骤自动化，哪些步骤人工判断。
- 作者如何处理失败。
- 是否有收费平台依赖。
- 是否能迁移到我们的工作台。

成片类视频：
- 镜头长度。
- 景别变化。
- 角色一致性。
- 动作强度。
- 转场方式。
- 台词密度。
- 音效点。
- 哪些镜头可以作为模板。

失败案例：
- 静态漂移。
- 角色跑偏。
- 动作弱。
- 画面崩坏。
- 武器/服装错误。
- 剪辑节奏差。

## 产出格式

每个视频输出一份审阅笔记：

```text
视频名：
来源：
总时长：
核心流程：
关键帧观察：
可迁移能力：
不能直接照搬的部分：
对工作台的改造建议：
```

## 对工作台规划的意义

逐帧审阅最终要反哺三个模块：

1. 项目模板：把优秀教程里的步骤变成可点击流程。
2. 质检规则：把失败案例变成自动检查项。
3. 自动剪辑：把优秀成片的镜头节奏变成剪辑策略。
