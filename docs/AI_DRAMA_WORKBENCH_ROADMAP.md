# AI短剧工作台路线

这个仓库以 AIComicBuilder 作为第一版地基，目标不是只做一个黑神话吕布项目，而是沉淀成后续长期可复用的 AI 短剧、漫剧、分镜、角色资产、配音、成片生产工作台。

## 第一阶段：跑通地基

- 保留 AIComicBuilder 的核心流程：剧本导入、角色管理、四视图参考、智能分镜、首尾帧、视频提示词、Seedance/Kling/Veo 视频生成、字幕与合成。
- 本地使用 SQLite 保存项目、角色、镜头、素材、任务状态。
- 上传素材统一保存在 `uploads`，数据库保存在 `data/aicomic.db`。

## 第二阶段：做成你的专用短剧流水线

- 增加项目模板：电影横屏短剧、竖屏短剧、漫剧、角色设定集、分镜资产包。
- 增加镜头质检：检查静态画面、角色跑偏、参考图缺失、动作不成立、风格不统一、比例错误。
- 增加“逐镜头五秒片段”模式：每个镜头独立生成，方便后期剪辑和二次筛选。
- 增加素材包导入：角色三视图、全身图、武器、对手、杂兵、场景、特效、标题卡、片尾卡统一入库。

## 第三阶段：接近 1人漫剧 / Flova 的桌面工作台

- 用 Electron 或 Tauri 包成桌面端，打开就是“AI短剧工作台”。
- 增加供应商中心：Seedance、可灵、Veo、OpenAI、Gemini、火山方舟、ComfyUI、本地模型。
- 增加流程画布：素材节点、提示词节点、视频节点、质检节点、合成节点可以连线。
- 增加项目记忆：每个项目保存世界观、角色禁忌、镜头语言、审美参考、常用负面词。

## Codex 协作模式

- 工作台负责批量生成、素材管理、任务执行、视频合成。
- Codex 负责搭流程、改提示词、修失败、做审美质检、调整代码和生产策略。
- 目标是让用户只需要提供素材、参考效果和最终审美判断，其余流程逐步交给工作台和 Codex 协同完成。

## 可选外部画布：LibTV

- LibTV CLI 可作为外部画布自动化工具，用于创建画布、上传素材、创建节点和分组。
- 默认只做素材同步和画布搭建，不自动触发可能消耗额度的生成任务。
- 后续可增加“导出到 LibTV”能力：一个镜头对应一个 LibTV 分组，镜头素材、提示词、视频节点自动铺好。

## 黑神话吕布项目要补的能力

- 角色资产：吕布魔神形态、三名强敌、杂兵、武器、坐骑或战场标志物。
- 场景资产：虎牢关/下邳/白门楼等战役背景可选，优先做成大规模战场空间。
- 动作资产：一挑三、方天画戟横扫、冲阵、格挡、破甲、近景怒吼、慢动作斩击。
- 特效资产：赤黑煞气、战场尘火、兵刃火星、冲击波、披风风压、血色天光。
- 声音资产：角色台词、旁白、战鼓、金属碰撞、低频冲击、片头片尾音乐。

## 判断标准

每个镜头生成后必须看三件事：角色是否像、动作是否动、镜头是否有电影感。只要其中一个不达标，就不进入最终合成。
