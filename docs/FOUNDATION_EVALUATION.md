# AI短剧工作台地基评估

结论：可以用 AIComicBuilder 作为第一版地基继续做。它已经具备剧本、角色、分镜、首尾帧、参考图、视频生成、字幕合成和多供应商配置，比从零开始更适合你后续长期做多个短剧项目。

## 综合判断

推荐程度：8/10

适合继续做的原因：
- 已经有完整的短剧生产链路，不只是单个网页工具。
- 数据结构包含项目、分集、角色、镜头、镜头资产、任务队列、提示词模板，后续可扩展。
- 已接入 Seedance、UCloud Seedance、Kling、Veo、Wan 等视频供应商框架。
- 支持 16:9、9:16、1:1，适合横屏电影版和竖屏短剧共存。
- 已有首尾帧模式和参考图模式，符合“每个镜头生成一个 5 秒片段，后期再剪”的生产方式。

主要短板：
- 产品入口仍偏技术项目，缺少小白引导和素材包工作流。
- 视频质检是弱检查，失败不会阻断流程，无法及时防止静态画面、角色跑偏、动作不足。
- 任务队列是单机轮询，能用但不够工业化，后期批量生成需要进度、失败原因、重试策略更清楚。
- 素材管理还没有形成“人物素材、三视图、全身图、对手、杂兵、场景、特效、声音”的统一资产库。
- 还不是桌面端，只是本地网页工作台。

## 与 1人漫剧的差距

1人漫剧优势大概率在产品包装、工作流模板、收费算力聚合、普通用户上手路径、精修体验和内置服务稳定性。

我们现在的优势是：
- 代码可控，可以按你的长期项目改。
- 可以接你自己的 Seedance、ComfyUI、火山、OpenAI、Gemini 等通道。
- 可以做黑神话吕布这类重资产项目模板，而不是被固定流程限制。
- 可以逐步沉淀成你自己的短剧资产库和提示词风格库。

## 下一阶段优先级

P0：先把它变成你能稳定使用的工作台。
- 首页与导航改成 AI短剧工作台。
- 增加一键启动脚本。
- 明确项目模板：横屏电影短剧、竖屏短剧、漫剧、角色资产包。

P1：解决“图不动、跑偏、质感不够”的核心痛点。
- 增加逐镜头 5 秒生成模式。
- 增加镜头质检：角色一致性、运动强度、镜头语言、参考图使用情况。
- 生成失败或质量不足时给出明确原因和下一步建议。
- 增加自动剪辑前置检查：只让合格片段进入最终合成。

P2：补素材资产库。
- 人物：主角、对手、杂兵、武器、服装、表情、动作。
- 场景：战场、室内、远景、近景、毁坏状态、气氛光。
- 特效：冲击波、火星、尘土、血色天光、披风风压。
- 声音：台词、旁白、BGM、打击音、环境音。

P3：桌面端。
- 用 Electron 或 Tauri 包装本地网页。
- 加启动器、项目目录选择、日志窗口、模型配置向导。
- 后续再做画布节点模式。

## 对黑神话吕布项目的落地判断

可落地，但不要直接让视频模型自由发挥。正确方式是：

1. 先建角色资产包：吕布魔神形态、三名强敌、杂兵、武器、战场。
2. 每个镜头绑定明确参考图。
3. 每个镜头只生成 5 秒左右，动作写死，镜头写死。
4. 每个片段生成后先质检，不达标不进入合成。
5. 合成阶段再加片头、片尾、字幕、BGM 和转场。

判断标准只有三条：角色像不像，动作动不动，镜头有没有电影感。
